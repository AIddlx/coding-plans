## 产品概述

腾讯云 Coding Plan 是为 AI Coding 场景推出的专属订阅套餐，支持多个主流模型，适配多种编程工具和"龙虾"（Agent）工具。

### 核心特点

- **多模型集成**: 支持 Tencent HY 2.0、GLM-5、Kimi-K2.5、MiniMax-M2.5 等 7+ 个主流模型
- **双场景覆盖**: 同时支持 **"龙虾"（Agent）场景** 和 **AI 编程场景**
- **工具生态丰富**: 支持 9+ 个编程工具 + 5+ 个"龙虾"工具（工具总数业界最多）
- **性价比高**: 相比 API 按量计费便宜 50%-80%
- **灵活套餐**: 2 档套餐（Lite/Pro），满足不同需求

## 快速开始

### 步骤1: 订阅套餐

1. 访问 [腾讯云Coding Plan购买页](https://buy.cloud.tencent.com/hunyuan)
2. 选择 Lite 或 Pro 套餐
3. 完成支付（新用户享受特惠价格）

### 步骤2: 获取API Key和Base URL

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 [Coding Plan页面](https://hunyuan.cloud.tencent.com/#/app/subscription)
3. 获取**专属API Key**（格式为 `sk-sp-xxxx`）
4. 记录对应的 Base URL（见下文）

### 步骤3: 配置Base URL

根据工具选择正确的Base URL：

| 协议类型 | Base URL |
|----------|----------|
| **OpenAI兼容协议** | `https://api.lkeap.cloud.tencent.com/coding/v3` |
| **Anthropic兼容协议** | `https://api.lkeap.cloud.tencent.com/coding/anthropic` |

> **重要**: Coding Plan 专属的 API Key 和 Base URL 与腾讯云预付费及后付费的 API Key（`sk-xxxx`）和 Base URL（`https://api.lkeap.cloud.tencent.com/xxxxx/`）**不互通**，请勿混用。

### 步骤4: 配置工具（以Claude Code为例）

1. 打开Claude Code设置
2. 添加 Custom Provider
3. 填入：
   - API Key: `sk-sp-xxxx`（腾讯云Coding Plan专属）
   - Base URL: `https://api.lkeap.cloud.tencent.com/coding/anthropic`
   - Model: 选择模型名（如 `hunyuan-2.0-instruct` 或 `glm-5`）
4. 保存配置并开始使用

### 步骤5: 监控用量

- 访问 [用量统计](https://hunyuan.cloud.tencent.com/#/app/subscription) 查看 5小时/每周/每月额度消耗
- 注意：额度用完后调用会失败，**不会自动转按量计费**

## 使用条件与限制

### ✅ 允许的使用场景

- 在支持的编程工具中使用（Claude Code、Cursor、CodeBuddy等）
- 正常的编码辅助任务（代码生成、补全、解释、重构等）
- 在"龙虾"（Agent）工具中使用（OpenClaw、WorkBuddy等）

### ❌ 禁止的使用场景

- **API 调用**: 不得用于自动化脚本、自定义应用后端、非交互式批量调用
- **账号共享**: 套餐为订阅人专享，**严禁账号共享**
- **多账号限制**: 同一主账号及其子账号**只能购买一个套餐**，共用额度
- **违规后果**: 可能导致订阅暂停或 API Key 封禁

### 账号规范

- 套餐仅限订阅人个人使用
- 团队协作需购买第三方大模型 OpenAI 兼容接口（后付费模式）
- 同一账号不能购买多个套餐

## 常见问题

### Q1: 腾讯云Token Plan和Coding Plan是什么关系？

**A**: 腾讯云将产品命名为"Token Plan"，但实际计费模式为**API请求次数**（与传统Coding Plan一致），而非Token数量。这是命名上的差异，实质相同。

### Q2: 如何选择Lite还是Pro？

- **Lite**: 18,000次/月 ≈ 70轮问答，适合新手体验、轻度使用
- **Pro**: 90,000次/月 ≈ 5倍Lite额度，适合重度开发者、多仓库并行

### Q3: 额度用完后怎么办？

额度用完后调用会失败，**不会自动转按量计费**。解决方案：
1. 升级到更高套餐
2. 等待下月额度刷新
3. 避免额度耗尽影响工作

### Q4: 为什么感觉并发只有1？

套餐的并发限制与套餐等级相关：**Pro > Lite**。但实际并发可能受以下因素影响：
- 高峰期集群流量大时可能动态限流
- 如命中风控规则可能被限流/封禁
- 具体以实际使用为准

### Q5: 购买后可以退款吗？

**不支持退款**。订阅前请仔细阅读使用须知，确认符合使用条件。
